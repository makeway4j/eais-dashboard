# Source Policy

## Preferred Sources

Use direct and reputable sources first:

- Official AI vendor blogs, release notes, docs, and status pages.
- Government and regulator pages.
- Standards bodies and AI safety institutes.
- Company press releases for data center and infrastructure deals.
- SEC filings and investor relations pages.
- Reputable news outlets for confirmation and context.

## Handling Unverified Items

If an item is not well confirmed:

- Mark it as `watch`.
- Include the source link.
- Avoid strong claims.
- Do not present speculation as fact.

## Required Metadata

Every item should keep:

- `title`
- `source`
- `url`
- `publishedAt`
- `topic`
- `importance`
- `summary`
- `whyItMatters`

## Deduping

When multiple sources cover the same event:

1. Prefer the primary source for factual details.
2. Use reputable reporting for context.
3. Link both only when each adds value.
